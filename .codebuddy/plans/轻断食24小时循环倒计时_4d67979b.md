## 用户核心需求

修改轻断食倒计时逻辑，实现24小时循环倒计时：

- 断食阶段（如16小时）结束后，自动切换到进食阶段（如8小时）倒计时
- 界面右侧显示模式名称（如"16:8 轻断食"）和状态文字（"断食中"或"觅食中"）上下排列
- 圆环中间的"剩余"文字需要根据阶段动态显示

## 四种轻断食模式

| 模式 | 断食时长 | 进食时长 |
| --- | --- | --- |
| 16:8 | 16小时 | 8小时 |
| 18:6 | 18小时 | 6小时 |
| 20:4 | 20小时 | 4小时 |
| OMAD | 23小时 | 1小时 |


## 功能内容

1. 断食阶段倒计时结束后，自动切换到进食阶段倒计时
2. 进食阶段倒计时结束后，自动切换回断食阶段（或提示用户）
3. 界面根据当前阶段显示不同文案："断食中"或"觅食中"
4. 进度条颜色区分：断食阶段用绿色，进食阶段用橙色
5. 按钮文案根据阶段变化：断食中显示"提前进食"，进食中显示"结束进食"

## Tech Stack Selection

- 现有技术栈：微信小程序 + JavaScript + Less
- 无需引入新技术，基于现有代码修改

## Implementation Approach

### 核心逻辑修改

**1. 修改 `utils/fasting.js` 的 `getCurrentState()` 函数**

现有问题：`getCurrentState()` 只计算当前阶段剩余时间，不会自动切换阶段。

修改方案：

- 检测 `remaining <= 0` 时，自动切换到下一阶段
- 断食阶段结束 → 自动切换到进食阶段
- 进食阶段结束 → 自动切换回断食阶段（开始新的循环）

**2. 修改 `pages/overview/index.js` 的计时器逻辑**

现有问题：计时器检测到 `remaining <= 0` 时只是清除计时器和震动，不会自动切换阶段。

修改方案：

- 移除 `remaining <= 0` 时的清除计时器逻辑
- 保留震动提示
- 让 `getCurrentState()` 自动处理阶段切换

**3. 修改界面显示逻辑**

- 圆环中间的"剩余"文字改为根据阶段显示："断食剩余"或"觅食剩余"
- 右侧状态文字根据阶段显示："断食中，已进行XX:XX:XX"或"觅食中，已进行XX:XX:XX"
- 进度条颜色根据阶段变化

### 关键代码修改

**utils/fasting.js - getCurrentState() 修改：**

```javascript
function getCurrentState() {
  const state = storage.get(storage.KEYS.FASTING_STATE, null);
  if (!state || state.status === STATUS.IDLE) {
    return { status: STATUS.IDLE, plan: null };
  }

  const plan = getPlan(state.planId);
  const now = Date.now();
  const end = new Date(state.endTime).getTime();
  const start = new Date(state.startTime).getTime();
  const total = end - start;
  const elapsed = now - start;
  const remaining = Math.max(0, end - now);
  const progress = total > 0 ? Math.min(1, Math.max(0, elapsed / total)) : 0;

  // 自动切换阶段
  if (remaining <= 0 && state.status !== STATUS.IDLE) {
    // 震动提示
    wx.vibrateShort({ type: 'light' });
    
    // 自动切换到下一阶段
    const fastHours = state.customFastHours || plan.fastHours;
    const eatHours = state.customEatHours || plan.eatHours;
    
    const newNow = new Date();
    const newNowStr = newNow.toISOString();
    
    if (state.status === STATUS.FASTING) {
      // 断食结束，切换到进食
      const newEnd = new Date(newNow.getTime() + eatHours * 3600 * 1000);
      const newState = {
        ...state,
        status: STATUS.EATING,
        startTime: newNowStr,
        endTime: newEnd.toISOString(),
      };
      storage.set(storage.KEYS.FASTING_STATE, newState);
      
      return {
        ...newState,
        plan,
        progress: 0,
        remaining: eatHours * 3600 * 1000,
        remainingText: formatDuration(eatHours * 3600 * 1000),
        elapsed: 0,
        elapsedText: '00:00:00',
        totalText: formatDuration(eatHours * 3600 * 1000),
        isOverdue: false,
      };
    } else {
      // 进食结束，切换回断食
      const newEnd = new Date(newNow.getTime() + fastHours * 3600 * 1000);
      const newState = {
        ...state,
        status: STATUS.FASTING,
        startTime: newNowStr,
        endTime: newEnd.toISOString(),
        currentCycleStart: newNowStr,
      };
      storage.set(storage.KEYS.FASTING_STATE, newState);
      
      return {
        ...newState,
        plan,
        progress: 0,
        remaining: fastHours * 3600 * 1000,
        remainingText: formatDuration(fastHours * 3600 * 1000),
        elapsed: 0,
        elapsedText: '00:00:00',
        totalText: formatDuration(fastHours * 3600 * 1000),
        isOverdue: false,
      };
    }
  }

  return {
    ...state,
    plan,
    progress,
    remaining,
    remainingText: formatDuration(remaining),
    elapsed,
    elapsedText: formatDuration(elapsed),
    totalText: formatDuration(total),
    isOverdue: now > end,
  };
}
```

**pages/overview/index.js - startTimerIfNeeded() 修改：**

```javascript
startTimerIfNeeded() {
  if (this._timer) return;
  if (this.data.fastingState.status === 'idle') return;

  this._timer = setInterval(() => {
    const state = fasting.getCurrentState();
    this.setData({ fastingState: state });
    // 移除清除计时器的逻辑，让 getCurrentState() 自动处理阶段切换
  }, 1000);
},
```

### 性能与可靠性

- 自动阶段切换逻辑在 `getCurrentState()` 中处理，确保每次获取状态时都是最新的
- 震动提示使用 `wx.vibrateShort({ type: 'light' })`，轻量级不阻塞
- 阶段切换时更新本地存储，确保小程序重启后状态正确

## Implementation Notes

### 性能考虑

- `getCurrentState()` 每次调用都会检查阶段是否结束，性能开销极小
- 阶段切换时只更新必要的状态字段，避免不必要的重渲染

### 向后兼容

- 现有 `toggleState()` 和 `stopPlan()` 函数保持不变
- 只是增强了 `getCurrentState()` 的自动切换能力

### 边界情况处理

- 如果用户在进食阶段点击"结束进食"，应该切换回断食阶段
- 如果用户在断食阶段点击"提前进食"，应该切换到进食阶段（现有逻辑已处理）
- 小程序意外关闭后重新打开，应该能正确恢复到当前阶段
